Voici mon setup informatique pour que tu aies le contexte :

MACHINES :
- guidesk : PC fixe Linux Mint, AMD Ryzen 5, RTX 3060, 32Go RAM. Machine principale.
- guilaptop : PC portable Ubuntu, tourne en permanence, héberge Home Assistant dans Docker + Tailscale.
- Connexion : 4G via routeur ASUS (CGNAT). IP LAN guilaptop : 192.168.50.234. Tailscale : 100.105.110.66.

HOME ASSISTANT :
- Installé sur guilaptop dans Docker
- Accès LAN : 192.168.50.234:8123 | Tailscale : 100.105.110.66:8123
- Partage SMB : //192.168.50.234/home-guilaptop (monté automatiquement sur guidesk via systemd)
- Point de montage guidesk : /mnt/guilaptopHA/

SAUVEGARDES (guidesk) :
- BCK-1 : copie manuelle 000_IMPORTANTs via script Nemo (#001-synchro_importantst.sh)
- BCK-2 : rsync horodaté tous les 7 démarrages via backup_check_gui.sh + backup_run_gui.sh (systemd user)
  → Config : ~/.config/backup/config.conf | Counter : ~/.config/backup/counter
  → Option HA : si /mnt/guilaptopHA monté → propose rsync HA → 2To
  → Option Mega : mega_sync.sh → 000_IMPORTANTs + HA vers Mega (rclone)
- Disque 2To : /media/guidesktop/2To/000.backup/
- Mega : rclone remote "mega:" — ~/.local/bin/mega_sync.sh
- Règle 3-2-1 : ✓ CONFORME (3 copies, 2 supports, 1 hors site Mega)

DONNÉES CRITIQUES :
- 000_IMPORTANTs : conteneur VeraCrypt 13Go dans /home/guidesktop/Documents/
- KeePass dans le conteneur VeraCrypt
- Photos smartphone : conteneur VeraCrypt 20Go sur 2To (backup Google Photos)

// Montage SMB guilaptop ne fonctionne pas 
Le montage SMB de guilaptop ne fonctionne pas sur guidesk. Voici le setup :
- Service : /etc/systemd/system/mnt-guilaptopHA.mount
- Point de montage : /mnt/guilaptopHA/
- Partage : //192.168.50.234/home-guilaptop (LAN) ou //100.105.110.66/home-guilaptop (Tailscale)
- Credentials : /etc/samba/credentials-guilaptop (chmod 600)
- Type : cifs, vers=3.0

Commandes de diagnostic :
  systemctl status mnt-guilaptopHA.mount
  ping -c 3 192.168.50.234
  smbclient -L //192.168.50.234 -N
  sudo mount -t cifs //192.168.50.234/home-guilaptop /mnt/guilaptopHA -o credentials=/etc/samba/credentials-guilaptop,uid=1000,gid=1000,vers=3.0
